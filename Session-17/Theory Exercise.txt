1.)Πως μπορούμε να κάνουμε ένα Request σε ένα εξωτερικό API χωρίς να γνωρίζει ολόκληρο το url μας;

-Μην έχοντας ολόκληρο το url,κοιτάμε το API documentation.Θα βρούμε στοιχεία για την δημιουργία ενος request,διεύθυνση URL,request body και headers. 

2.)Αν κάνω ενα GET request και ζητήσω ενα jpeg image, το response τι 
content-type πρεπει να εχει;

-Πρέπει σε κάθε περίπτωση να έχει (image/jpeg).

3.)Πως μπορώ να κάνω ένα GET request και το περιεχόμενο του να κρατηθεί στην cache για 1 μέρα;

-Στο cache-control header set max-age=τιμή ή Expire header set σε συγκεκριμένη ημερομηνία.

4.)Μπορώ να έχω μια εφαρμογή που κάνει μόνο GET requests;

-Βεβαίως,το GET αποτελεί τρόπο λήψης δεδομένων,κάτι που μπορεί να συμβεί σε ενα απλό browser.

5.)Μπορώ να έχω μια εφαρμογή που κάνει μόνο POST requests; 

-Και αυτό γίνεται,δημιουργώντας μια εφαρμογή που στέλνει δεδομένα σε ένα Server και κανει update ενα log με το ημερίσιο report.

6.)Τι status code περιμένω από ένα API αν δεν έχω στείλει ένα υποχρεωτικό πεδίο;

-Λόγω μη αντίληψης και συνεπώς αδυναμίας εκτέλεσης,θα εμφανίσει "400 Bad Request".

7.)Τι status code περιμένω από ένα API αν δεν υπάρχει επικοινωνία με τη βάση;

-Αν η βάση είναι offline και ο Server δεν μπορεί να επικοινωνήσει,θα εμφανίσει error code "500" κάτι μη αναμενόμενο συνέβη.

8.)Μπορώ να κάνω Login με GET request;

-Είναι εφικτό να πραγματοποιηθεί η σύνδεση,αλλά για λόγους ασφαλείας και για την διασφάλιση των προσωπικών δεδομενών (GDPR),δεν συνίσταται,καθώς θα είναι φανερά τα στοιχεία σύνδεσης(Username,Password) στο URL.

9.)Ποια είναι η διαφορά ανάμεσα στο PUT και στο POST;

- a)Η μέθοδος PUT ενημερώνει τα ήδη υπάρχοντα δεδομένα.
  b)Η μέθοδος POST θα ζητήσει απο τον Server να δημιουργήσει νέα δεδομένα.

10.)Ποιά μέθοδο θα πρέπει να χρησιμοποιήσω για να «ανεβάσω» ένα αρχείο;

-Πραγματοποιείται η μέθοδος POST.
